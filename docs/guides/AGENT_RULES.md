# Agent Operating Rules

> CRITICAL: All agents MUST read and follow these rules before any operation.

---

## 1. Documentation Standards

### Rule 1.1: Bilingual Structure
All documentation MUST be bilingual (English and Traditional Chinese). The document structure must follow the "Full English followed by Full Chinese" format. First, write the complete English content of the document. Then, insert a clear horizontal separator (`---`). Finally, provide the complete Traditional Chinese translation.

### Rule 1.2: No Inline Mixing
Do NOT mix English and Chinese within the same section or paragraph. The transition between languages should only happen once via the separator.

### Rule 1.3: Consistency
The Chinese translation must mirror the structure, headings, and formatting of the English version exactly. If the English section has a table, the Chinese section must have the same table with translated content.

---

## 2. Code Standards

### Rule 2.1: Comments
All code comments MUST be bilingual. Write the English comment first, followed by the Chinese translation on the next line.

### Rule 2.2: Docstrings
All function/class docstrings MUST include both English and Chinese descriptions. Write the full English documentation within the docstring, then the full Chinese documentation.

---

## 3. Naming Conventions

### Rule 3.1: File Names
Use English for all file and directory names.

### Rule 3.2: Variable Names
Use descriptive English names for variables, functions, and classes.

---

## 4. Error Messages

### Rule 4.1: Bilingual Errors
All error messages MUST be displayed in both languages. English first, then Chinese on a new line.

---

## 5. User Interface

### Rule 5.1: CLI Output
All CLI output MUST be bilingual.

---

## Enforcement
These rules are MANDATORY. Any agent or skill that violates these rules should be flagged for review and correction.

---

# Agent 運作規則

> 重要：所有 Agent 在執行任何操作前都必須閱讀並遵守這些規則。

---

## 1. 文檔標準

### 規則 1.1：雙語結構
所有文檔都必須是雙語的（英文和繁體中文）。文檔結構必須遵循「整篇英文接整篇中文」的格式。首先，編寫文檔的完整英文內容。然後，插入一個清晰的水平分隔線（`---`）。最後，提供完整的繁體中文翻譯。

### 規則 1.2：不要行內混合
不要在同一章節或段落中混合英文和中文。語言之間的轉換只能透過分隔線發生一次。

### 規則 1.3：一致性
中文翻譯必須完全對應英文版本的結構、標題和格式。如果英文部分有表格，中文部分必須有內容經過翻譯的相同表格。

---

## 2. 程式碼標準

### 規則 2.1：註解
所有程式碼註解都必須是雙語的。先寫英文註解，下一行則寫中文翻譯。

### 規則 2.2：文檔字串
所有函數/類別的文檔字串都必須包含英文和中文描述。在文檔字串中先寫完整的英文說明，再寫完整的中文說明。

---

## 3. 命名規範規則

### 規則 3.1：檔案名稱
所有檔案和目錄名稱均使用英文。

### 規則 3.2：變數名稱
變數、函數和類別使用具描述性的英文名稱。

---

## 4. 錯誤訊息

### 規則 4.1：雙語錯誤
所有錯誤訊息都必須以雙語顯示。先英文，後中文。

---

## 5. 使用者介面

### 規則 5.1：命令列輸出
所有命令列輸出都必須是雙語的。

---

## 執行
這些規則是強制性的。任何違反這些規則的 Agent 或技能都應該被標記以供審查和修正。
